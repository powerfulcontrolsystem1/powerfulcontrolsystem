Carrito unificado por empresa

Empresa: cargando...

El sistema usará una sola página de carrito para todas las estaciones. Aquí defines qué bloques estarán activos por defecto para toda la empresa; luego puedes sobrescribirlos por estación desde Configuración de estaciones.

Aumenta botones, campos, tarjetas de cobro y catalogo por botones para operar con el dedo en tablet, monitor POS o pantalla tactil.
Si está activo, el carrito no permite pagar hasta seleccionar o crear un cliente de la empresa.

Botones de acciones del carrito

Activa solo los botones que debe ver el cajero en la barra de acciones.

Si esta desactivado, el cajero no vera el check de alerta visual en las acciones del carrito.
Por defecto queda apagada; el cajero puede activarla manualmente si el check visible esta habilitado.
Cuando esta activo, el cajero puede cerrar la cuenta en modo sin internet; el sistema guarda la venta en este equipo y la sincroniza cuando vuelva la conexion, respetando la caja del usuario.
Por defecto el soporte impreso queda identificado como provisional/offline hasta que el sistema sincronice la venta.

Pagos por QR

Configura las cuentas receptoras donde llegara el dinero de Bre-B, Nequi u otro QR bancario.

El cobro se registra como transferencia bancaria con una referencia automatica; el cajero debe verificar el abono antes de pagar y cerrar.
Activa Nombre Proveedor Llave / cuenta Comercio Payload oficial / plantilla Instrucciones Acciones

Puedes usar placeholders en el payload oficial: {valor}, {referencia}, {llave}, {comercio}, {empresa_id}, {carrito_id}, {carrito_codigo}. Si el proveedor entrega un QR estatico o una plantilla EMV/API, pegala aqui.

Las estaciones individuales pueden heredar esta configuración o sobrescribirla desde la pantalla de configuración de estaciones.
