Carrito unificado por empresa

Empresa: cargando...

El sistema usará una sola página de carrito para todas las estaciones. Aquí defines qué bloques estarán activos por defecto para toda la empresa; luego puedes sobrescribirlos por estación desde Configuración de estaciones.

Respuesta de botones del carrito

Define si los botones del carrito emiten confirmacion sonora o vibracion visual/fisica al tocarlos.

En PC se aplica como vibracion visual breve del boton.
En celulares compatibles usa vibracion fisica y tambien una respuesta visual corta.

Atajos de teclado POS

Configura por empresa las teclas rapidas usadas por cajeros en supermercados, tiendas y puntos de venta.

ENTER solo agrega producto cuando el foco esta en el lector/codigo o cuando no estas escribiendo en otro campo. ALT+F4 sale del carrito hacia el panel del sistema si el navegador permite capturar la tecla.

Báscula electrónica

Activa esta opción solo si la empresa venderá productos por peso usando una báscula serial/USB compatible con Web Serial.

Por defecto queda apagada. Al activarla, el cajero podrá conectar el puerto local de la báscula y aplicar el peso a la cantidad del producto.
Si está activo, el carrito no permite pagar hasta seleccionar o crear un cliente de la empresa.
Se usa en carritos y comprobantes cuando no se elige un cliente registrado.

Botones de acciones del carrito

Activa solo los botones que debe ver el cajero en la barra de acciones.

Permite mover una cuenta abierta a otra mesa, habitacion o estacion disponible. En motel/hotel solo se permite si la tarifa destino es equivalente.
Si esta desactivado, el cajero no vera el check de alerta visual en las acciones del carrito.
Por defecto queda apagada; el cajero puede activarla manualmente si el check visible esta habilitado.

Medios de pago habilitados

Activa los medios que puede usar la empresa en el carrito. Bre-B queda listo para conciliacion automatica cuando se conecte el webhook o API bancaria.

Mientras no exista webhook bancario, el cajero debe registrar la referencia o llave confirmada.
Permite cerrar ventas contra el cupo aprobado del cliente y crear cartera por cobrar.
Cuando esta activo, el cajero puede cerrar la cuenta en modo sin internet; el sistema guarda la venta en este equipo y la sincroniza cuando vuelva la conexion, respetando la caja del usuario.
Por defecto el soporte impreso queda identificado como provisional/offline hasta que el sistema sincronice la venta.
Cuando la venta tenga CUFE/CUDE o codigo de validacion, el comprobante impreso agrega un QR de consulta publica de la factura electronica.

Pagos por QR

Configura las cuentas receptoras donde llegara el dinero de Bre-B, Nequi u otro QR bancario.

El cobro se registra como transferencia bancaria con una referencia automatica; el cajero debe verificar el abono antes de pagar y cerrar.
Activa Nombre Proveedor Llave / cuenta Comercio Payload oficial / plantilla Instrucciones Acciones

Puedes usar placeholders en el payload oficial: {valor}, {referencia}, {llave}, {comercio}, {empresa_id}, {carrito_id}, {carrito_codigo}. Si el proveedor entrega un QR estatico o una plantilla EMV/API, pegala aqui.

Las estaciones individuales pueden heredar esta configuración o sobrescribirla desde la pantalla de configuración de estaciones.
