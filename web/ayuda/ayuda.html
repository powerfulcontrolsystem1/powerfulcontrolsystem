Inventario y reportes

  • El inventario se administra por empresa con bodegas, movimientos y existencias separadas por empresa_id.
  • El módulo de reportes consolida ventas cerradas, compras de inventario y stock actual.
  • Los reportes deben consumirse siempre con empresa_id para no mezclar datos entre empresas.

Carrito y lector de código de barras

  • El carrito permite agregar productos por catálogo o por lectura de código de barras/SKU.
  • Al leer un código válido, el sistema incrementa cantidad o crea el item automáticamente.
  • El proceso descuenta inventario del producto asociado respetando el contexto de empresa.

Configuración por empresa

  • La configuración operativa se guarda por llave de empresa en la interfaz de administración.
  • La configuración avanzada/fiscal se guarda en base de datos por empresa_id.
  • No se debe reutilizar una configuración de una empresa en otra sin respaldo/restauración explícita.
