Ayuda — Chat con IA

Cómo usarlo

  • Escribe una pregunta (ej: “muéstrame los últimos 10 carritos pagados”).
  • La IA responde con explicación y, si hace falta ejecutar algo, mostrará una tarjeta “Acciones sugeridas”.
  • Para ejecutar, debes presionar Confirmar y ejecutar. Nada se ejecuta automáticamente.

Acciones (PCS_ACTION) y confirmaciones

  • GET/OPEN: consultar datos o abrir una vista del sistema.
  • POST/PUT: crear o actualizar datos (siempre pide confirmación).
  • DELETE: eliminar datos (pide confirmación adicional escribiendo ELIMINAR).

Recomendación: pide primero una consulta (GET) para revisar resultados y luego confirma cambios.

Chat IA empresarial (por empresa)

  • Opera solo sobre la empresa seleccionada (aislamiento por empresa_id).
  • Puede proponer CRUD por empresa usando el endpoint /api/empresa/db_admin (tablas con empresa_id).
  • Recibe contexto de auditoria reciente y consultas seguras resueltas por backend; la IA no ejecuta SQL libre.
  • Solo usuarios con rol administrador pueden ejecutar operaciones de base de datos/CRUD desde el chat.

Todas las operaciones quedan registradas en auditoría empresarial.

Chat IA global (super)

  • Sirve para consultas globales del sistema y navegación a vistas.
  • Cuando proponga acciones, siempre requiere confirmación.

Ejemplos útiles

  • “Lista las tablas disponibles para mi empresa y dime cuáles son las más importantes.”
  • “Busca el producto ‘Coca Cola’ y actualiza el precio a 3500 (muéstrame antes el producto encontrado).”
  • “Crea un código de descuento del 10% llamado PROMO10 y dime cómo desactivarlo después.”

Voz, robot y secretaria

  • La configuracion del chat flotante se administra por empresa desde Administrar empresa > Configuracion.
  • El administrador puede desactivar el chat/robot, elegir chat cuadrado, robot IA o secretaria IA, y activar el modo voz.
  • La secretaria usa voz femenina automaticamente; el robot conserva la voz elegida.
  • Si el servidor de voz streaming esta apagado o falla, el chat cae a voz del navegador o texto sin romper el servidor.

Trazabilidad

Las ejecuciones confirmadas desde el chat se registran con metadatos del chat (fuente chat_ia y conversación), y en operaciones de BD se registra además un evento en empresa_auditoria_eventos.
